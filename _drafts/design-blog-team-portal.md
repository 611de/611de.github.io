# 从零设计一个团队导航门户：Orbit 风格工作台的诞生过程

> 记录一次内部工具导航页的设计迭代，从需求梳理到三版方案对比的完整过程。

## 需求背景

团队需要一个**日常导航入口页**，作为成员每天打开浏览器后的第一个页面。核心需求：

- 快速访问常用工具和联系人
- 查看 CI/CD 构建状态
- 了解团队工作动态和网站使用数据
- 链接到各类内部系统和文档

本质上是一个**信息密集型的内部工作台**，需要在有限空间内高效展示多种信息类型。

## 第一版：三栏卡片网格

初始方案采用传统的**顶栏搜索 + 三栏网格**布局：

```
┌──────────────────────────────────────────────┐
│  Logo    搜索栏              日期  头像       │
├──────────────────────────────────────────────┤
│  KPI 卡片 × 4（访问量/成员/工单/部署）        │
├────────────┬─────────────┬───────────────────┤
│  常用工具   │  团队成员    │  使用数据 + 活动   │
│  (8个图标)  │  (8人头像)   │  (图表+时间线)     │
│            │             │                   │
│  CI/CD工具  │  工作进展    │                   │
│  (状态列表) │  (5条记录)   │                   │
├────────────┴─────────────┴───────────────────┤
│  页脚：帮助文档 · 提交反馈 · 系统状态          │
└──────────────────────────────────────────────┘
```

**设计决策：**

- 品牌色选用蓝色系（#2563eb），清爽专业
- 卡片使用 12px 圆角 + 极轻阴影（alpha ≤ 0.05），干净不厚重
- 三栏等宽布局，桌面/平板/手机自适应响应

**问题**：虽然功能齐全，但视觉上偏"后台管理"风格，缺少一点内部工具该有的亲和力和辨识度。

## 第二版：Orbit 风格 —— 侧栏 + 紫色品牌

参考了一张 Orbit 概念设计图后，决定转向**深色侧栏 + 浅色内容区**的工作台布局。

### 布局重构

```
┌────────┬─────────────────────────────────────┐
│ Orbit  │  工作空间 / 概览                      │
│        │  🔍 搜索栏          ⌘K  [+ 新建]    │
│ 工作空间 ├─────────────────────────────────────┤
│ · 概览  │  早上好, Yuan 👋                      │
│ · 项目  │  这是团队今天的工作概况                │
│        │  ┌──────┬──────┬──────┬──────┐       │
│ 工具中心 │  │访问量 │活跃数 │构建率 │待处理 │       │
│ · CI/CD │  └──────┴──────┴──────┴──────┘       │
│ · 动态  │  ┌────────────────┬─────────────┐   │
│ · 工具  │  │  常用工具 (4×2) │  团队动态    │   │
│        │  │  Figma/Notion/  │  头像堆叠    │   │
│ 资源    │  │  Slack/Jira... │  活动时间线  │   │
│ · 文档  │  ├────────────────┤  快速操作    │   │
│ · 看板  │  │  CI/CD 状态    │  2×2 按钮   │   │
│ · 设置  │  │  3条流水线      │             │   │
│        │  └────────────────┴─────────────┘   │
│ [Y] Yuan│                                     │
└────────┴─────────────────────────────────────┘
```

### 关键改动

| 维度 | 第一版 | 第二版 |
|------|--------|--------|
| 导航结构 | 顶部横栏 | 左侧深色侧栏（220px） |
| 品牌色 | 蓝色 #2563eb | 紫色 #6366f1 |
| 内容区比例 | 三栏等宽 | 左 3/5 + 右 2/5 |
| 搜索体验 | 简单搜索框 | 圆角搜索栏 + ⌘K 快捷键 |
| 个性化 | 无 | 时段问候语（早上好/下午好/晚上好） |
| 工具展示 | 图标列表 | 4×2 网格卡片 |
| CI/CD | 状态列表 | 独立卡片 + 运行中呼吸动画 |

### 为什么选紫色？

- 蓝色在企业工具中太常见，容易和竞品混淆
- 紫色（#6366f1）在深色侧栏上对比度更好
- 传达"创新、效率"的调性，符合内部工具的定位

## 第三版：完整导航门户

在 Orbit 风格基础上，新增三个核心模块：

### 新增模块

**1. 常用联系人**

5 位核心联系人，每人显示：
- 彩色首字母头像
- 姓名 + 角色描述（如"技术负责人 · 后端"）
- 在线状态指示（绿色在线 / 黄色忙碌 / 灰色离线）
- 快捷消息按钮

```
┌─────────────────────────┐
│  📇 常用联系人            │
│                         │
│  [张] 张三         🟢 [💬] │
│      技术负责人 · 后端    │
│                         │
│  [李] 李四         🟢 [💬] │
│      前端负责人 · React   │
│  ...                    │
└─────────────────────────┘
```

**2. 常用链接**

6 个高频内部系统入口，每项带 Lucide 图标：
- API 文档、数据库管理、权限管理平台
- 测试环境、周报模板、公司内网首页

**3. CI/CD 常用链接**

6 个 DevOps 工具直达入口：
- Jenkins、GitHub Actions、ArgoCD
- SonarQube、Docker Registry、Grafana

每个工具用**品牌色首字母图标** + **域名提示**展示，辨识度高。

## 设计原则总结

### 配色系统

```css
/* 品牌色 */
--color-brand: #6366f1;          /* 紫色主色 */
--color-brand-light: #818cf8;    /* 浅紫 */
--color-brand-bg: #eef2ff;       /* 极浅紫背景 */

/* 侧栏 */
--sidebar-bg: #0f172a;           /* 深海军蓝 */

/* 中性色 */
--color-bg-secondary: #f8fafc;   /* 内容区浅灰底 */
--color-border: #e2e8f0;         /* 卡片边框 */

/* 状态色 */
--state-success: #16a34a;        /* 在线/成功 */
--state-warning: #ca8a04;        /* 忙碌/警告 */
--state-error: #dc2626;           /* 离线/失败 */
```

### 阴影与圆角

- 卡片阴影 alpha 值 ≤ 0.05，几乎无感
- 静态卡片优先用边框分隔，不依赖阴影
- 圆角统一 12px（卡片）、8px（按钮）、4px（标签）

### 响应式策略

| 断点 | 侧栏 | 内容区 |
|------|------|--------|
| ≥ 1024px | 完整 220px 侧栏 | 多栏网格 |
| 768px–1023px | 缩为 60px 图标栏 | 双栏 |
| < 768px | 隐藏（汉堡菜单） | 单栏堆叠 |

## 技术实现

- HTML + Tailwind CDN（无构建步骤）
- CSS 变量统一管理 design token
- Lucide 图标库
- 纯内联 JS 实现时段问候和日期显示
- 所有工具品牌色和头像色定义为 CSS class，避免内联样式

## 最终效果

三版方案对比：

| 版本 | 风格 | 适合场景 |
|------|------|----------|
| V1 团队导航门户 | 蓝色三栏网格 | 简洁信息展示 |
| V2 Orbit 风格 | 紫色侧栏工作台 | 日常高频使用 |
| V3 完整导航门户 | Orbit + 联系人/链接/CI | 全功能入口页 |

**推荐 V3**作为最终方案——它在 Orbit 的视觉基础上补全了实际工作中最常用的联系人、链接和 CI/CD 工具入口，是一个真正可以投入使用的团队导航门户。
